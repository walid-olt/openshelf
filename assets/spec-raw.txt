Context

Une bibliothèque possède un catalogue contenant des centaines d'ouvrages. Afin de faciliter la gestion de ce catalogue, il est nécessaire de disposer d'une application permettant d'ajouter, modifier, consulter et supprimer des livres.

Dans ce projet, vous allez développer une application Full Stack avec Next.js.

L'objectif principal n'est pas de créer un système complet de gestion de bibliothèque mais de découvrir les principales fonctionnalités Full Stack offertes par Next.js :

    Les Route Handlers
    Les opérations CRUD
    La communication Frontend ↔ Backend
    MongoDB avec Mongoose
    La validation des données avec Zod
    Les routes dynamiques
    La navigation entre les pages

Toutes les données devront être stockées dans une base MongoDB.
Fonctionnalités détaillées

1 - Page d'accueil — Catalogue des livres

La page d'accueil représente le catalogue de la bibliothèque.

Elle doit afficher :

    Le nom de la bibliothèque
    Une courte description
    La liste complète des livres

Chaque livre doit afficher :

    Le titre
    L'auteur
    La catégorie
    L'année de publication
    Son statut (Disponible / Emprunté)
    Un bouton Voir détails
    Un bouton Modifier
    Un bouton Supprimer

Les données doivent être récupérées depuis la base de données via une Route Handler.

2 - Ajouter un livre

Créer une page /books/create

Cette page contient un formulaire permettant d'ajouter un nouveau livre.

Chaque livre contient :

    Titre
    Auteur
    ISBN
    Catégorie
    Année de publication
    Description

Toutes les données doivent être validées avant leur enregistrement.

Après création, l'utilisateur est redirigé vers la page d'accueil.

3 - Détail d'un livre

Créer une route dynamique : /books/[id]

Cette page affiche toutes les informations du livre :

    Titre
    Auteur
    ISBN
    Catégorie
    Année
    Description
    Statut

4 - Modifier un livre

Créer une page : /books/edit/[id]

Le formulaire doit être pré-rempli avec les informations du livre.

Après modification, l'utilisateur est redirigé vers la page de détail du livre.

5 - Supprimer un livre

Depuis la liste des livres, l'utilisateur peut supprimer un ouvrage.

Une confirmation doit être demandée avant la suppression.

Après suppression, la liste est automatiquement mise à jour.

6 - Recherche

Ajouter une barre de recherche permettant de rechercher un livre par :

    titre
    auteur

La recherche peut être réalisée côté client.

7 - Filtrage

Ajouter un filtre permettant d'afficher :

    Tous
    Disponible
    Emprunté

8 - Responsive Design

L’ensemble des pages de l’application doit être responsive et mobile-friendly.

L’interface doit être adaptée aux résolutions suivantes :

    Mobile (≥ 375px)
    Tablette (≥ 768px)
    Desktop (≥ 1024px)

API REST

Les Route Handlers suivants doivent être implémentés.

Livres

GET     /api/books

GET     /api/books/:id

POST    /api/books

PUT     /api/books/:id

DELETE  /api/books/:id

Chaque Route Handler devra retourner des réponses HTTP appropriées.
Validation

Toutes les données doivent être validées avec Zod.

Les règles minimales sont :

Titre:

    obligatoire
    minimum 3 caractères

Auteur:

    obligatoire

ISBN:

    obligatoire
    unique

Catégorie:

    obligatoire

Année:

    nombre valide

Description:

    minimum 10 caractères

Les erreurs doivent être affichées à l'utilisateur.
Gestion de la base de données

Utiliser MongoDB avec Mongoose.

Créer un modèle Book contenant les propriétés suivantes :

    title
    author
    isbn
    category
    publicationYear
    description
    available
    createdAt
    updatedAt

Consignes techniques

Layout global

Créer un layout partagé contenant :

Header

    Nom de la bibliothèque
    Lien Accueil
    Lien Ajouter un livre

Footer

    Nom de l'application
    Année courante

Le Header et le Footer doivent être visibles sur toutes les pages.
Communication Frontend ↔ Backend

Toutes les opérations CRUD doivent passer par les Route Handlers.

Les accès directs à MongoDB depuis les composants React sont interdits.
Gestion des erreurs

Afficher un message utilisateur dans les cas suivants :

    livre introuvable
    erreur de validation
    erreur serveur

Navigation

Utiliser exclusivement le composant Link de Next.js.

Les redirections après création, modification ou suppression doivent être correctement gérées.
Bonus

1 — Tri alphabétique

    Permettre de trier les livres par titre.

2 — Tri par année

    Du plus récent au plus ancien.

3 — Pagination

    Afficher les livres par groupes de 10.

Contraintes techniques

    Utiliser TypeScript sur tous les fichiers
    Aucun fichier JavaScript accepté
    Aucun type any
    Utiliser MongoDB
    Utiliser Mongoose
    Utiliser Zod pour la validation
    Utiliser les Route Handlers de Next.js
    Respecter une architecture modulaire
    Les composants doivent être réutilisables
